# e-Gov 法令 API (Version 1) 仕様書

> **原典:** デジタル庁「e-Gov 法令 API 仕様書（Version 1）」1.4 版（2024-07-29）および「別紙1 法令 API メッセージ一覧」  
> **本文書の範囲:** REST API としてのエンドポイント仕様・パラメータ・レスポンス構造・エラー体系に焦点を当てている。応答 XML の要素定義（XSD 等）および法令本文の XML スキーマ（LawFullText 内部構造）は別途提供される仕様を参照のこと。

---

## 1. 概要

e-Gov 法令 API は、日本の公布済み現行法令のデータを取得するための REST API である。利用にあたり申込みは不要で、誰でも利用できる。

### 1.1. ベース URL

```
https://laws.e-gov.go.jp/api/{Version}/
```

本仕様書が対象とする Version は **`1`** である。URI 中の `{Version}` には `1` を設定すること。

> **変更履歴:** 1.4 版（2024-07-29）にて、e-Gov 法令検索リニューアルに伴いホスト名が `elaws.e-gov.go.jp` から `laws.e-gov.go.jp` に変更された。

### 1.2. 共通仕様

- **HTTP メソッド:** 全 API とも `GET`
- **レスポンス形式:** XML（`<?xml version="1.0" encoding="UTF-8"?>`）
- **認証:** 不要

### 1.3. 応答 XML の共通構造

すべての API の応答 XML は以下の 2 ブロックで構成される。

```xml
<?xml version="1.0" encoding="UTF-8"?>
<DataRoot>
  <Result>
    <Code>...</Code>       <!-- 処理結果コード -->
    <Message>...</Message>  <!-- エラー時のメッセージ。正常時は空文字列 -->
  </Result>
  <ApplData>
    <!-- API ごとのデータ項目 -->
  </ApplData>
</DataRoot>
```

- `<Result>` は常に出力される。
- `<ApplData>` はシステム例外発生時には出力されない。それ以外の場合（エラーコード 1 を含む）は出力される。

### 1.4. XML エスケープ処理

応答 XML では以下の文字がエスケープされている。利用側で必要に応じて復元すること。

| 処理前文字 | エスケープ後 |
|-----------|----------|
| `'`       | `&apos;` |
| `"`       | `&quot;` |
| `&`       | `&amp;`  |
| `<`       | `&lt;`   |
| `>`       | `&gt;`   |

## 3. 法令名一覧取得 API

公布済み現行法令の法令 ID、名称、法令番号および公布年月日の一覧を取得する。法令種別を指定して絞り込むことも可能。

### 3.1. リクエスト

```
GET https://laws.e-gov.go.jp/api/1/lawlists/{法令種別}
```

### 3.2. URI パラメータ

| パラメータ | 必須 | 型 | 説明 |
|----------|------|-----|------|
| `Version` | ○ | 英数字 | API バージョン番号。`1` を指定。 |
| `法令種別` | ○ | 英数字 | 取得対象の法令種別コード。[法令種別コード表](#81-法令種別コード)を参照。 |

### 3.3. レスポンス

#### HTTP ステータスコード

| コード | 意味 |
|------|------|
| `200 OK` | 正常 |
| `400 Bad Request` | パラメータ不正等、利用者起因エラー |
| `404 Not Found` | 該当データなし |
| `500 Internal Server Error` | サーバ内部エラー |

#### ApplData の構造

| 要素 | 出現回数 | 型 | 説明 |
|------|----------|-----|------|
| `Category` | 1 | 英数字（1桁） | 取得要求した法令種別コード |
| `LawNameListInfo` | 0..n | 複合 | 法令名一覧情報。取得件数 0 件の場合はタグ自体が出力されない。 |

`LawNameListInfo` の子要素:

| 要素 | 出現回数 | 型 | 説明 |
|------|----------|-----|------|
| `LawId` | 1 | 半角（1文字以上） | 法令 ID |
| `LawName` | 1 | 全角（1文字以上） | 法令名称 |
| `LawNo` | 1 | 全半角（1文字以上） | 法令番号 |
| `PromulgationDate` | 1 | 数字（8桁） | 公布年月日（`yyyyMMdd` 形式） |

### 3.4. リクエスト例

```
GET https://laws.e-gov.go.jp/api/1/lawlists/2
```

→ 憲法・法律の一覧を取得

## 4. 法令取得 API

施行中の現行法令の全文を取得する。本文に図が含まれる場合は画像データ（Base64 エンコード済み ZIP）も取得可能。

### 4.1. リクエスト

```
GET https://laws.e-gov.go.jp/api/1/lawdata/{法令番号 or 法令ID}
```

### 4.2. URI パラメータ

| パラメータ | 必須 | 型 | 説明 |
|----------|------|-----|------|
| `Version` | ○ | 英数字 | API バージョン番号。`1` を指定。 |
| `法令番号` | △ | 全半角 | 取得対象の法令番号（例: `平成十五年法律第五十七号`） |
| `法令ID` | △ | 半角 | 取得対象の法令 ID（例: `415AC0000000057`） |

> **△:** 法令番号または法令 ID のいずれか一方が必須。

### 4.3. レスポンス

#### HTTP ステータスコード

| コード | 意味 |
|------|------|
| `200 OK` | 正常 |
| `400 Bad Request` | パラメータ不正等、利用者起因エラー |
| `404 Not Found` | 該当データなし |
| `406 Not Acceptable` | 返却可能な容量超過、または法令データが複数存在 |
| `500 Internal Server Error` | サーバ内部エラー |

#### ApplData の構造

| 要素 | 出現回数 | 型 | 説明 |
|------|----------|-----|------|
| `LawId` | 1 | 半角（0 or 1文字以上） | 取得要求した法令 ID |
| `LawNum` | 1 | 全半角（0 or 1文字以上） | 取得要求した法令番号 |
| `LawFullText` | 1 | 全半角（1文字以上） | 該当法令の全文 XML |
| `ImageData` | 0..1 | 英数字（1文字以上） | 画像情報。画像をフォルダ名 `pict` に収集し、フォルダごと ZIP 圧縮したファイルを Base64 エンコードした値。法令全文に画像を含む場合のみ出力。 |

> **注:** `LawId`・`LawNum` は、リクエストで指定しなかった側が空文字列（0桁）となる。

### 4.4. リクエスト例

法令番号で取得:
```
GET https://laws.e-gov.go.jp/api/1/lawdata/平成十五年法律第五十七号
```

法令 ID で取得:
```
GET https://laws.e-gov.go.jp/api/1/lawdata/415AC0000000057
```

## 5. 条文内容取得 API

法令番号（または法令 ID）と条・項・別表の組合せを指定して、該当する条文の内容を取得する。

### 5.1. リクエスト

```
GET https://laws.e-gov.go.jp/api/1/articles;lawNum={法令番号};article={条};paragraph={項};appdxTable={別表}
```

または法令 ID を使用する場合:

```
GET https://laws.e-gov.go.jp/api/1/articles;lawId={法令ID};article={条};paragraph={項};appdxTable={別表}
```

> **注:** パラメータはセミコロン `;` で区切るマトリクスパラメータ形式である（クエリ文字列の `?` / `&` 形式ではない）。

### 5.2. URI パラメータ

| パラメータ名 | URI 上の名称 | 必須 | 型 | 説明 |
|-----------|-----------|------|-----|------|
| Version | - | ○ | 英数字 | API バージョン番号。`1` を指定。 |
| 法令番号 | `lawNum` | △ | 全半角 | 取得対象の法令番号 |
| 法令 ID | `lawId` | △ | 半角 | 取得対象の法令 ID |
| 条 | `article` | - | 全半角 | 取得対象の条（例: `第十一条`） |
| 項 | `paragraph` | - | 全半角 | 取得対象の項 |
| 別表 | `appdxTable` | - | 全半角 | 取得対象の別表 |

> **△:** 法令番号または法令 ID のいずれか一方が必須。

### 5.3. パラメータ設定パターン

条・項・別表の指定には以下の有効な組合せがある。パターンに合致しない指定はエラーとなる。

| 取得パターン | 法令番号/法令ID | 条 (`article`) | 項 (`paragraph`) | 別表 (`appdxTable`) |
|------------|--------------|------|------|--------|
| **条を取得** | ○ | ○ | - | - |
| **項を取得**（条の指定なし） | ○ | - | ○ | - |
| **条配下の項を取得** | ○ | ○ | ○ | - |
| **別表を取得** | ○ | - | - | ○ |

### 5.4. レスポンス

#### HTTP ステータスコード

| コード | 意味 |
|------|------|
| `200 OK` | 正常 |
| `300 Multiple Choices` | 別表取得時に複数の候補が存在 |
| `400 Bad Request` | パラメータ不正等、利用者起因エラー |
| `404 Not Found` | 該当データなし |
| `406 Not Acceptable` | 返却可能な容量超過、または法令データが複数存在 |
| `500 Internal Server Error` | サーバ内部エラー |

#### 処理結果コード（条文内容取得 API 固有）

この API では `Result/Code` に `2`（複数候補あり）が返される場合がある。他の API では `0` または `1` のみ。

#### ApplData の構造

| 要素 | 出現回数 | 型 | 説明 |
|------|----------|-----|------|
| `LawId` | 1 | 半角（0 or 1文字以上） | 取得要求した法令 ID |
| `LawNum` | 1 | 全半角（0 or 1文字以上） | 取得要求した法令番号 |
| `Article` | 1 | 全半角（0 or 1文字以上） | 取得要求した条。項のみ取得・別表取得の場合は空。 |
| `Paragraph` | 1 | 全半角（0 or 1文字以上） | 取得要求した項。条のみ取得・別表取得の場合は空。 |
| `AppdxTable` | 1 | 全半角（0 or 1文字以上） | 取得要求した別表。条・項の取得の場合は空。 |
| `LawContents` | 1 | 全半角（1文字以上） | 指定条件に合致する条文の内容 XML |
| `AppdxTableTitleLists` | 0..1 | 複合 | 別表名の候補リスト。処理結果コードが `2` の場合のみ出力。 |
| `ImageData` | 0..1 | 英数字（1文字以上） | 画像情報（法令取得 API と同仕様）。該当条文に画像を含む場合のみ出力。 |

`AppdxTableTitleLists` の子要素:

| 要素 | 出現回数 | 型 | 説明 |
|------|----------|-----|------|
| `AppdxTableTitle` | 1..n | 全半角（1文字以上） | 条件に合致した別表名の候補 |

### 5.5. リクエスト例

法令番号を指定して条を取得:
```
GET https://laws.e-gov.go.jp/api/1/articles;lawNum=平成十五年法律第五十七号;article=第十一条
```

法令 ID を指定して条を取得:
```
GET https://laws.e-gov.go.jp/api/1/articles;lawId=415AC0000000057;article=第十一条
```

### 5.6. 補足事項

- **法令番号重複:** 法令番号が重複している法令は、`lawNum` パラメータでは取得できない。`lawId` を使用すること。
- **別表名が長すぎる場合:** URI パラメータに指定する別表名が長すぎると `Request Rejected` エラー HTML が返される。別表名は**前方一致**で検索されるため、別表名を短縮して再実行すること。（[Request Rejected について](#9-留意事項)を参照）

## 6. 更新法令一覧取得 API

指定した日付に更新された法令の一覧を取得する。取得項目は e-Gov 法令検索の更新法令一覧画面から出力できる CSV ファイルと同等。

### 6.1. リクエスト

```
GET https://laws.e-gov.go.jp/api/1/updatelawlists/{年月日}
```

### 6.2. URI パラメータ

| パラメータ | 必須 | 型 | 説明 |
|----------|------|-----|------|
| `Version` | ○ | 英数字 | API バージョン番号。`1` を指定。 |
| `年月日` | ○ | 数字（8桁） | 取得対象の日付。`yyyyMMdd` 形式。 |

> **制約:** 指定可能な年月日は **2020年11月24日（`20201124`）以降**。未来の日付は指定不可。

### 6.3. レスポンス

#### HTTP ステータスコード

| コード | 意味 |
|------|------|
| `200 OK` | 正常 |
| `400 Bad Request` | パラメータ不正等、利用者起因エラー |
| `404 Not Found` | 該当データなし |
| `500 Internal Server Error` | サーバ内部エラー |

#### ApplData の構造

| 要素 | 出現回数 | 型 | 説明 |
|------|----------|-----|------|
| `Date` | 1 | 数字（1桁以上） | 取得要求した年月日 |
| `LawNameListInfo` | 0..n | 複合 | 更新日に合致する法令名一覧情報。取得件数 0 件の場合はタグ自体が出力されない。 |

`LawNameListInfo` の子要素:

| 要素 | 出現回数 | 型 | 説明 |
|------|----------|-----|------|
| `LawTypeName` | 1 | 全角（1文字以上） | 法令種別名 |
| `LawNo` | 1 | 全半角（1文字以上） | 法令番号 |
| `LawName` | 1 | 全角（1文字以上） | 法令名称 |
| `LawNameKana` | 1 | 全角（1文字以上） | 法令名読み（かな） |
| `OldLawName` | 1 | 全角（1文字以上） | 旧法令名 |
| `PromulgationDate` | 1 | 数字（8桁） | 公布年月日（`yyyyMMdd` 形式） |
| `AmendName` | 1 | 全角（1文字以上） | 改正法令名 |
| `AmendNo` | 1 | 全角（1文字以上） | 改正法令番号 |
| `AmendPromulgationDate` | 1 | 全角（1文字以上） | 改正法令公布日 |
| `EnforcementDate` | 1 | 全角（1文字以上） | 施行日 |
| `EnforcementComment` | 1 | 全半角（1文字以上） | 施行日備考 |
| `LawId` | 1 | 半角（1文字以上） | 法令 ID |
| `LawUrl` | 1 | 半角（1文字以上） | 本文 URL |
| `EnforcementFlg` | 1 | 半角（1桁） | 未施行フラグ。`0`: 施行済、`1`: 未施行 |
| `AuthFlg` | 1 | 半角（1桁） | 所管課確認フラグ。`0`: 確認済、`1`: 確認中 |

### 6.4. リクエスト例

```
GET https://laws.e-gov.go.jp/api/1/updatelawlists/20241001
```

## 7. 処理結果コード

応答 XML の `Result/Code` に設定される値。

| コード | 名称 | 説明 |
|-------|------|------|
| `0` | 正常 | 正常終了 |
| `1` | エラー | 取得件数 0 件の場合、およびパラメータ指定された法令番号等に該当する法令データが複数存在する場合も含む |
| `2` | 複数候補あり | 条文内容取得 API で別表を取得する場合に、複数候補が存在する場合のみ設定 |

## 9. 留意事項

### 9.1. 別表取得時の複数候補

条文内容取得 API で別表を取得する際、`appdxTable` パラメータの指定によっては複数の別表が返却対象候補となる場合がある。この場合、HTTP ステータス `300 Multiple Choices` が返され、`Result/Code` は `2` となり、`AppdxTableTitleLists` に候補リストが格納される。

候補リストから対象の別表名を特定し、`appdxTable` パラメータを再設定して改めて API を実行すること。

### 9.2. 法令番号の重複

法令番号が重複している法令が存在する。このような法令は法令取得 API・条文内容取得 API で `lawNum` を使用した取得ができない（`406 Not Acceptable` またはエラーメッセージ No.9 が返される）。`lawId` を使用すること。

### 9.3. Request Rejected

各 API 実行時に、正常な XML ではなく以下のような HTML がレスポンスされる場合がある。

```html
<html>
<head><title>Request Rejected</title></head>
<body>
リクエスト内容を表示できません。お手数ですが、以下の Your support ID に続く数字を
電子政府利用支援センターまでお知らせください。<br>
The requested URL was rejected. ...
<br><br>Your support ID is: *******************
</body>
</html>
```

**原因:** リクエスト URI のパラメータ値が不正、または別表名が長すぎる場合。別表名は前方一致検索のため、短縮して再試行可能。

**対処:**
1. リクエストパラメータの値を見直す。
2. 解消しない場合は、送信日時と Your support ID を控え、[電子政府利用支援センター](https://www.e-gov.go.jp/contact/form/enquete.html)に問い合わせる。

## 10. エラーメッセージ一覧

応答 XML の `Result/Message` に設定されるメッセージの一覧。

| No. | メッセージ内容 | 法令名一覧取得 | 法令取得 | 条文内容取得 | 更新法令一覧取得 |
|-----|-------------|:-----------:|:------:|:---------:|:------------:|
| 1 | パラメータがありません。パラメータをご確認ください。 | ○ | ○ | | |
| 2 | 法令種別が誤っています。パラメータをご確認ください。 | ○ | | | |
| 3 | 法令番号又は法令IDが未設定又は誤っています。パラメータをご確認ください。 | | ○ | | |
| 4 | 条、項又は別表が未設定又は誤っています。パラメータをご確認ください。 | | | ○ | |
| 5 | 取得結果が０件でした。取得条件に誤りがないかご確認ください。 | ○ | ○ | ○ | ○ |
| 6 | 取得条件に指定した条、項、別表に該当する条文内容が存在しません。 | | | ○ | |
| 7 | 複数の候補が存在します。取得条件に誤りがないかご確認ください。 | | | ○ | |
| 8 | 取得条件に該当する別表の候補を複数検出しました。別表名をご確認ください。 | | | ○ | |
| 9 | 指定された法令番号に該当する法令データが複数件存在します。該当法令データの取得には、法令データの一括ダウンロードをご利用ください。 | | ○ | ○ | |
| 10 | 日付が誤っています。パラメータをご確認ください。 | | | | ○ |

### メッセージ補足

- **No.1:** パラメータが指定されていない場合。
- **No.2:** 法令種別コードに `1`〜`4` 以外の値が指定された場合。
- **No.3:** 法令取得 API で法令番号・法令 ID がいずれも未設定の場合。
- **No.4:** 条文内容取得 API でパラメータ設定パターン（§5.3）に合致しない場合。
- **No.5:** 取得条件に合致する法令が存在しない場合。全 API 共通。
- **No.6:** 法令は存在するが、指定した条・項・別表に該当する条文が存在しない場合。
- **No.7:** 別表以外のパラメータ指定で複数候補が存在する場合。
- **No.8:** 別表取得時に複数の別表が候補となった場合（処理結果コード `2`）。
- **No.9:** 法令番号重複により複数の法令データが該当する場合。`lawId` を使用するか、一括ダウンロードを利用すること。
- **No.10:** 更新法令一覧取得 API で年月日パラメータが日付として認識できない場合。
